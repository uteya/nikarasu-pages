claude-code-action

@claude の使い方

基本

@claude は GitHub の Issue コメントや PR コメントに書くことで Claude Code を呼び出す仕組み。リポジトリに設定された GitHub Actions ワークフロー（claude-code-action）が自動で起動し、ブランチ作成・ファイル編集・PR 作成まで行う。

Issue の本文に書いても、コメント欄に書いてもOK。PR のレビューコメントでも使える。

依頼の書き方のコツ

  • 具体的に書く — 「資料を作って」より「Q1の売上データを表形式でまとめて」
  • ファイル名を指定する — 「docs/report.md に書いて」と出力先を明示
  • 1つの依頼に1つのタスク — 複数タスクは別 Issue に分ける方が精度が上がる
  • 修正は PR コメントで — PR のコメント欄に @claude で修正を依頼できる

実行例

例 1: レポート作成
@claude docs/report.md にQ1の売上データを表形式でまとめて。前年比も入れてください。
指定ファイルにデータを構造化して書き込む
例 2: 仕様書の修正
@claude この仕様書の「対象ユーザー」セクションを中小企業向けに修正して。具体的なペルソナを2つ追加してください。
既存ファイルの特定セクションを編集する
例 3: ステータス更新
@claude .context/CHANGES.md に「2026-03-09: GTM戦略書v4.0レビュー完了」を追記して。
変更履歴への追記（CHANGES.md は全員が追記可能）
例 4: 調査依頼
@claude docs/research/ に中小企業DX市場の調査レポートを作成して。市場規模、主要プレイヤー、トレンドをカバーしてください。
新規ファイルを作成して調査結果をまとめる
Project Structure

プロジェクトの見方

プロジェクトの状態は 3 つのファイルで把握できる。全て GitHub Web 上で閲覧可能。

.context/STATUS.md
プロジェクトの現在の状態。フェーズ、進捗率、メンバー別タスク、次のマイルストーンが記載されている。
読み方: セッション開始時にまず確認。自分の担当タスクと期限を把握する。
PM管理 — 小竹が更新。他メンバーは閲覧のみ。
.context/CHANGES.md
変更履歴。いつ、誰が、何をしたかが時系列で記録されている。
読み方: 前回から何が変わったか確認する。自分の作業完了時にも追記する。
全員追記可 — Append-Only。既存エントリは消さない。
docs/decisions.md
意思決定ログ。なぜその判断をしたか、根拠チェーン付きで記録される。
読み方: 過去の判断理由を知りたいときに参照する。「なぜこうなっている？」の答えがここにある。
Rules

守ってほしいルール（5つだけ）

  1. 1
    STATUS.md は PM（小竹）が管理 — 編集しない プロジェクトの状態は PM が責任を持つ。参照は自由。
  2. 2
    CHANGES.md は追記のみ — 既存エントリを消さない 履歴は積み重ね。誰の記録も消さないこと。
  3. 3
    PR にはレビュアーを指定する 作成したPRには相手をレビュアーに設定。内容確認を経てからマージ。
  4. 4
    直接 main に push しない — PR 経由で main は保護ブランチ。全ての変更は PR を経由する。
  5. 5
    困ったら Issue で相談 判断に迷うこと、ブロッカー、疑問は全て Issue に書く。記録が残り、後から参照できる。
Phase 2

Desktop App が必要になったら

いつ必要になるか

以下のケースに該当する場合、ローカルに Claude Code Desktop App を導入する:

  • • ローカルファイルの操作が必要（大量のファイル編集、ビルド作業など）
  • • MCP 連携を使いたい（Google Calendar、Slack、Backlog 等）
  • • 複雑な Claude Code セッションを実行したい
  • • Skills や Hooks を活用したい

セットアップ概要

詳細手順は claude-bootstrap/guides/team-collaboration.md の「Phase 2: Desktop App」を参照。

Windows
winget install Anthropic.ClaudeCode
Mac
brew install claude-code

インストール後: リポジトリ clone → CLAUDE.local.md 作成 → claude 起動。npm（npm install -g @anthropic-ai/claude-code）でもインストール可能。

FAQ

よくある質問

以下を確認してください:
1. リポジトリに .github/workflows/claude-code.yml が存在するか
2. GitHub リポジトリの Settings → Secrets に ANTHROPIC_API_KEY が設定されているか
3. Actions タブでワークフローが有効になっているか

解決しない場合は Issue で小竹に相談してください。
PR のコメント欄に @claude [修正内容] と書けば、Claude が修正を加えてコミットします。具体的に書くほど精度が上がります。
例: @claude 表の3列目を「前年比」から「成長率(%)」に変更して
GitHub Web 上でファイルを開き、右上の鉛筆アイコン（Edit this file）をクリックすれば直接編集できます。編集後は「Commit changes」で保存。

ただし main ブランチに直接コミットせず、「Create a new branch for this commit and start a pull request」を選択して PR 経由でマージしてください。
複数ファイルにまたがる大きな変更や、ビルド・テストが必要な作業は Desktop App の導入を検討してください。

セットアップ手順は上記「Phase 2: Desktop App が必要になったら」セクションを参照。Windows: winget install Anthropic.ClaudeCode / Mac: brew install claude-code でインストールできます。
